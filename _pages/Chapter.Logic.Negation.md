## Constructive negation

In constructive logic, the `⊥` data type has a fundamental role
since it allows us to define negation. Showing that the *negation*
of a proposition `A` holds amounts to showing that a proof of `A`
can be turned into a proof of `⊥`.

<pre class="Agda"><a id="¬_"></a><a id="746" href="Chapter.Logic.Negation.html#746" class="Function Operator">¬_</a> <a id="749" class="Symbol">:</a> <a id="751" href="Agda.Primitive.html#388" class="Primitive">Set</a> <a id="755" class="Symbol">→</a> <a id="757" href="Agda.Primitive.html#388" class="Primitive">Set</a>
<a id="761" href="Chapter.Logic.Negation.html#746" class="Function Operator">¬</a> <a id="763" href="Chapter.Logic.Negation.html#763" class="Bound">A</a> <a id="765" class="Symbol">=</a> <a id="767" href="Chapter.Logic.Negation.html#763" class="Bound">A</a> <a id="769" class="Symbol">→</a> <a id="771" href="Data.Empty.html#914" class="Function">⊥</a>
</pre>
Using negation in conjunction with propositional equality we can
define the notion of "being different from", thus:

<pre class="Agda"><a id="_≢_"></a><a id="899" href="Chapter.Logic.Negation.html#899" class="Function Operator">_≢_</a> <a id="903" class="Symbol">:</a> <a id="905" class="Symbol">∀{</a><a id="907" href="Chapter.Logic.Negation.html#907" class="Bound">A</a> <a id="909" class="Symbol">:</a> <a id="911" href="Agda.Primitive.html#388" class="Primitive">Set</a><a id="914" class="Symbol">}</a> <a id="916" class="Symbol">→</a> <a id="918" href="Chapter.Logic.Negation.html#907" class="Bound">A</a> <a id="920" class="Symbol">→</a> <a id="922" href="Chapter.Logic.Negation.html#907" class="Bound">A</a> <a id="924" class="Symbol">→</a> <a id="926" href="Agda.Primitive.html#388" class="Primitive">Set</a>
<a id="930" href="Chapter.Logic.Negation.html#930" class="Bound">x</a> <a id="932" href="Chapter.Logic.Negation.html#899" class="Function Operator">≢</a> <a id="934" href="Chapter.Logic.Negation.html#934" class="Bound">y</a> <a id="936" class="Symbol">=</a> <a id="938" href="Chapter.Logic.Negation.html#746" class="Function Operator">¬</a> <a id="940" class="Symbol">(</a><a id="941" href="Chapter.Logic.Negation.html#930" class="Bound">x</a> <a id="943" href="Agda.Builtin.Equality.html#150" class="Datatype Operator">≡</a> <a id="945" href="Chapter.Logic.Negation.html#934" class="Bound">y</a><a id="946" class="Symbol">)</a>
</pre>
As an example, let us prove that `true` and `false` are
different. We start with a definition like this:

<pre class="Agda"><a id="true≢false₁"></a><a id="1063" href="Chapter.Logic.Negation.html#1063" class="Function">true≢false₁</a> <a id="1075" class="Symbol">:</a> <a id="1077" href="Agda.Builtin.Bool.html#198" class="InductiveConstructor">true</a> <a id="1082" href="Chapter.Logic.Negation.html#899" class="Function Operator">≢</a> <a id="1084" href="Agda.Builtin.Bool.html#192" class="InductiveConstructor">false</a>
<a id="1090" href="Chapter.Logic.Negation.html#1063" class="Function">true≢false₁</a> <a id="1102" class="Symbol">=</a> <a id="1104" class="Hole">{!!}</a>
</pre>
In order to make progress, recall that the type `true ≢ false` is
definitionally equal to the type `¬ (true ≡ false)` which, in turn,
is definitionally equal to the type `true ≡ false → ⊥`. That is,
`true≢false` is nothing but a *function* that accepts a proof of
`true ≡ false` and yields something of type `⊥`. We can obtain
evidence of this fact by giving a name --- say `p` --- to the
argument of the function:

<pre class="Agda"><a id="true≢false₂"></a><a id="1534" href="Chapter.Logic.Negation.html#1534" class="Function">true≢false₂</a> <a id="1546" class="Symbol">:</a> <a id="1548" href="Agda.Builtin.Bool.html#198" class="InductiveConstructor">true</a> <a id="1553" href="Chapter.Logic.Negation.html#899" class="Function Operator">≢</a> <a id="1555" href="Agda.Builtin.Bool.html#192" class="InductiveConstructor">false</a>
<a id="1561" href="Chapter.Logic.Negation.html#1534" class="Function">true≢false₂</a> <a id="1573" href="Chapter.Logic.Negation.html#1573" class="Bound">p</a> <a id="1575" class="Symbol">=</a> <a id="1577" class="Hole">{!!}</a>
</pre>
At this stage Agda expects us to fill the hole with a term of type
`⊥`. This is clearly impossible, but in the context we also have a
proof `p` that `true ≡ false`. If we inspect `p` using case
analysis, Agda figures out that there cannot be such proof and
replaces `p` with the absurd pattern. This way, we are freed from
the obligation to fill the goal.

<pre class="Agda"><a id="true≢false₃"></a><a id="1948" href="Chapter.Logic.Negation.html#1948" class="Function">true≢false₃</a> <a id="1960" class="Symbol">:</a> <a id="1962" href="Agda.Builtin.Bool.html#198" class="InductiveConstructor">true</a> <a id="1967" href="Chapter.Logic.Negation.html#899" class="Function Operator">≢</a> <a id="1969" href="Agda.Builtin.Bool.html#192" class="InductiveConstructor">false</a>
<a id="1975" href="Chapter.Logic.Negation.html#1948" class="Function">true≢false₃</a> <a id="1987" class="Symbol">()</a>
</pre>
Occasionally it is useful to define a function that accepts an
absurd pattern "on the spot". In these cases we can use the syntax
`λ ()` to define such function. For example, we can prove that
`true` is different from `false` also in the following way:

<pre class="Agda"><a id="true≢false₄"></a><a id="2253" href="Chapter.Logic.Negation.html#2253" class="Function">true≢false₄</a> <a id="2265" class="Symbol">:</a> <a id="2267" href="Agda.Builtin.Bool.html#198" class="InductiveConstructor">true</a> <a id="2272" href="Chapter.Logic.Negation.html#899" class="Function Operator">≢</a> <a id="2274" href="Agda.Builtin.Bool.html#192" class="InductiveConstructor">false</a>
<a id="2280" href="Chapter.Logic.Negation.html#2253" class="Function">true≢false₄</a> <a id="2292" class="Symbol">=</a> <a id="2294" class="Symbol">λ</a> <a id="2296" class="Symbol">()</a>
</pre>
We will see below examples where this notation is useful.

## Properties of negation

We will make a rather extensive use of negation in the following
chapters. For the time being, we prove a few laws related to
negation. The first one is **contradiction**, namely the fact that
if we have both a proof of `A` and a proof of `¬ A` then we can
obtain a proof of anything. Recalling that the negation of `A` is
defined as a function that turns a proof of `A` into a proof of `⊥`,
we see that contradiction simply amounts to function application and
an application of the elimination principle for `⊥`.

<pre class="Agda"><a id="contradiction"></a><a id="2909" href="Chapter.Logic.Negation.html#2909" class="Function">contradiction</a> <a id="2923" class="Symbol">:</a> <a id="2925" class="Symbol">∀{</a><a id="2927" href="Chapter.Logic.Negation.html#2927" class="Bound">A</a> <a id="2929" href="Chapter.Logic.Negation.html#2929" class="Bound">B</a> <a id="2931" class="Symbol">:</a> <a id="2933" href="Agda.Primitive.html#388" class="Primitive">Set</a><a id="2936" class="Symbol">}</a> <a id="2938" class="Symbol">→</a> <a id="2940" href="Chapter.Logic.Negation.html#2927" class="Bound">A</a> <a id="2942" class="Symbol">→</a> <a id="2944" href="Chapter.Logic.Negation.html#746" class="Function Operator">¬</a> <a id="2946" href="Chapter.Logic.Negation.html#2927" class="Bound">A</a> <a id="2948" class="Symbol">→</a> <a id="2950" href="Chapter.Logic.Negation.html#2929" class="Bound">B</a>
<a id="2952" href="Chapter.Logic.Negation.html#2909" class="Function">contradiction</a> <a id="2966" href="Chapter.Logic.Negation.html#2966" class="Bound">p</a> <a id="2968" href="Chapter.Logic.Negation.html#2968" class="Bound">n</a> <a id="2970" class="Symbol">=</a> <a id="2972" href="Data.Empty.html#1069" class="Function">⊥-elim</a> <a id="2979" class="Symbol">(</a><a id="2980" href="Chapter.Logic.Negation.html#2968" class="Bound">n</a> <a id="2982" href="Chapter.Logic.Negation.html#2966" class="Bound">p</a><a id="2983" class="Symbol">)</a>
</pre>
Recalling that in Agda the type `¬ A` is *defined* to be the same as
the type `A → ⊥`, the type of `contradiction` can also be
specialized to `∀{A : Set} → A → ¬ ¬ A`. This is one of the
so-called "double negation" laws.

<pre class="Agda"><a id="double-negation"></a><a id="3216" href="Chapter.Logic.Negation.html#3216" class="Function">double-negation</a> <a id="3232" class="Symbol">:</a> <a id="3234" class="Symbol">∀{</a><a id="3236" href="Chapter.Logic.Negation.html#3236" class="Bound">A</a> <a id="3238" class="Symbol">:</a> <a id="3240" href="Agda.Primitive.html#388" class="Primitive">Set</a><a id="3243" class="Symbol">}</a> <a id="3245" class="Symbol">→</a> <a id="3247" href="Chapter.Logic.Negation.html#3236" class="Bound">A</a> <a id="3249" class="Symbol">→</a> <a id="3251" href="Chapter.Logic.Negation.html#746" class="Function Operator">¬</a> <a id="3253" href="Chapter.Logic.Negation.html#746" class="Function Operator">¬</a> <a id="3255" href="Chapter.Logic.Negation.html#3236" class="Bound">A</a>
<a id="3257" href="Chapter.Logic.Negation.html#3216" class="Function">double-negation</a> <a id="3273" class="Symbol">=</a> <a id="3275" href="Chapter.Logic.Negation.html#2909" class="Function">contradiction</a>
</pre>
In classical logic, the inverse implication `¬ ¬ A → A` is also
assumed to be true. However, this implication is not provable in
constructive logic (it is instructive to **attempt** proving this
property).

Another interesting law concerning negation is **contraposition**,
asserting that if `A` implies `B`, then `¬ B` implies `¬ A`.

<pre class="Agda"><a id="contraposition"></a><a id="3634" href="Chapter.Logic.Negation.html#3634" class="Function">contraposition</a> <a id="3649" class="Symbol">:</a> <a id="3651" class="Symbol">∀{</a><a id="3653" href="Chapter.Logic.Negation.html#3653" class="Bound">A</a> <a id="3655" href="Chapter.Logic.Negation.html#3655" class="Bound">B</a> <a id="3657" class="Symbol">:</a> <a id="3659" href="Agda.Primitive.html#388" class="Primitive">Set</a><a id="3662" class="Symbol">}</a> <a id="3664" class="Symbol">→</a> <a id="3666" class="Symbol">(</a><a id="3667" href="Chapter.Logic.Negation.html#3653" class="Bound">A</a> <a id="3669" class="Symbol">→</a> <a id="3671" href="Chapter.Logic.Negation.html#3655" class="Bound">B</a><a id="3672" class="Symbol">)</a> <a id="3674" class="Symbol">→</a> <a id="3676" href="Chapter.Logic.Negation.html#746" class="Function Operator">¬</a> <a id="3678" href="Chapter.Logic.Negation.html#3655" class="Bound">B</a> <a id="3680" class="Symbol">→</a> <a id="3682" href="Chapter.Logic.Negation.html#746" class="Function Operator">¬</a> <a id="3684" href="Chapter.Logic.Negation.html#3653" class="Bound">A</a>
<a id="3686" href="Chapter.Logic.Negation.html#3634" class="Function">contraposition</a> <a id="3701" href="Chapter.Logic.Negation.html#3701" class="Bound">f</a> <a id="3703" href="Chapter.Logic.Negation.html#3703" class="Bound">p</a> <a id="3705" href="Chapter.Logic.Negation.html#3705" class="Bound">q</a> <a id="3707" class="Symbol">=</a> <a id="3709" href="Chapter.Logic.Negation.html#3703" class="Bound">p</a> <a id="3711" class="Symbol">(</a><a id="3712" href="Chapter.Logic.Negation.html#3701" class="Bound">f</a> <a id="3714" href="Chapter.Logic.Negation.html#3705" class="Bound">q</a><a id="3715" class="Symbol">)</a>
</pre>
Observe that we define `contraposition` as a function with three
arguments `f`, `p` and `q`, while its type appears to have only two
arguments, one of type `A → B` (that would be `f`) and the other of
type `¬ B` (that would be `p`). However, the type `¬ A` is actually
the type `A → ⊥`, so `contraposition` can be seen as also having a
third argument of type `A`, that would be `q`.

Using `contraposition` and `double-negation` we can prove that
*triple* negation implies *single* negation.

<pre class="Agda"><a id="triple-negation"></a><a id="4219" href="Chapter.Logic.Negation.html#4219" class="Function">triple-negation</a> <a id="4235" class="Symbol">:</a> <a id="4237" class="Symbol">∀{</a><a id="4239" href="Chapter.Logic.Negation.html#4239" class="Bound">A</a> <a id="4241" class="Symbol">:</a> <a id="4243" href="Agda.Primitive.html#388" class="Primitive">Set</a><a id="4246" class="Symbol">}</a> <a id="4248" class="Symbol">→</a> <a id="4250" href="Chapter.Logic.Negation.html#746" class="Function Operator">¬</a> <a id="4252" href="Chapter.Logic.Negation.html#746" class="Function Operator">¬</a> <a id="4254" href="Chapter.Logic.Negation.html#746" class="Function Operator">¬</a> <a id="4256" href="Chapter.Logic.Negation.html#4239" class="Bound">A</a> <a id="4258" class="Symbol">→</a> <a id="4260" href="Chapter.Logic.Negation.html#746" class="Function Operator">¬</a> <a id="4262" href="Chapter.Logic.Negation.html#4239" class="Bound">A</a>
<a id="4264" href="Chapter.Logic.Negation.html#4219" class="Function">triple-negation</a> <a id="4280" class="Symbol">=</a> <a id="4282" href="Chapter.Logic.Negation.html#3634" class="Function">contraposition</a> <a id="4297" href="Chapter.Logic.Negation.html#3216" class="Function">double-negation</a>
</pre>
## Decidability

In classical logic it is common to assume the validity of the
*excluded middle* principle, namely that `¬ A ⊎ A` is true for every
proposition `A`. As we know from the [previous
chapter](Chapter.Logic.Connectives.html), in constructive logic, a
proof of a disjunction `¬ A ⊎ A` embeds either a proof of `¬ A` or a
proof of `A`, hence it may very well be the case that we are unable
to prove `¬ A ⊎ A` if we cannot find a proof of `¬ A` nor a proof of
`A`. The propositions for which we are able to prove `¬ A ⊎ A` are
said to be **decidable**.

<pre class="Agda"><a id="Decidable"></a><a id="4884" href="Chapter.Logic.Negation.html#4884" class="Function">Decidable</a> <a id="4894" class="Symbol">:</a> <a id="4896" href="Agda.Primitive.html#388" class="Primitive">Set</a> <a id="4900" class="Symbol">→</a> <a id="4902" href="Agda.Primitive.html#388" class="Primitive">Set</a>
<a id="4906" href="Chapter.Logic.Negation.html#4884" class="Function">Decidable</a> <a id="4916" href="Chapter.Logic.Negation.html#4916" class="Bound">A</a> <a id="4918" class="Symbol">=</a> <a id="4920" href="Chapter.Logic.Negation.html#746" class="Function Operator">¬</a> <a id="4922" href="Chapter.Logic.Negation.html#4916" class="Bound">A</a> <a id="4924" href="Data.Sum.Base.html#625" class="Datatype Operator">⊎</a> <a id="4926" href="Chapter.Logic.Negation.html#4916" class="Bound">A</a>
</pre>
As an example of decidable property, consider the problem of
determining whether two boolean values are equal or not.  This can
be shown by considering all the possible cases, which are finite.

<pre class="Agda"><a id="Bool-eq-decidable"></a><a id="5132" href="Chapter.Logic.Negation.html#5132" class="Function">Bool-eq-decidable</a> <a id="5150" class="Symbol">:</a> <a id="5152" class="Symbol">∀(</a><a id="5154" href="Chapter.Logic.Negation.html#5154" class="Bound">x</a> <a id="5156" href="Chapter.Logic.Negation.html#5156" class="Bound">y</a> <a id="5158" class="Symbol">:</a> <a id="5160" href="Agda.Builtin.Bool.html#173" class="Datatype">Bool</a><a id="5164" class="Symbol">)</a> <a id="5166" class="Symbol">→</a> <a id="5168" href="Chapter.Logic.Negation.html#4884" class="Function">Decidable</a> <a id="5178" class="Symbol">(</a><a id="5179" href="Chapter.Logic.Negation.html#5154" class="Bound">x</a> <a id="5181" href="Agda.Builtin.Equality.html#150" class="Datatype Operator">≡</a> <a id="5183" href="Chapter.Logic.Negation.html#5156" class="Bound">y</a><a id="5184" class="Symbol">)</a>
<a id="5186" href="Chapter.Logic.Negation.html#5132" class="Function">Bool-eq-decidable</a> <a id="5204" href="Agda.Builtin.Bool.html#198" class="InductiveConstructor">true</a>  <a id="5210" href="Agda.Builtin.Bool.html#198" class="InductiveConstructor">true</a>  <a id="5216" class="Symbol">=</a> <a id="5218" href="Data.Sum.Base.html#700" class="InductiveConstructor">inj₂</a> <a id="5223" href="Agda.Builtin.Equality.html#207" class="InductiveConstructor">refl</a>
<a id="5228" href="Chapter.Logic.Negation.html#5132" class="Function">Bool-eq-decidable</a> <a id="5246" href="Agda.Builtin.Bool.html#198" class="InductiveConstructor">true</a>  <a id="5252" href="Agda.Builtin.Bool.html#192" class="InductiveConstructor">false</a> <a id="5258" class="Symbol">=</a> <a id="5260" href="Data.Sum.Base.html#675" class="InductiveConstructor">inj₁</a> <a id="5265" class="Symbol">λ</a> <a id="5267" class="Symbol">()</a>
<a id="5270" href="Chapter.Logic.Negation.html#5132" class="Function">Bool-eq-decidable</a> <a id="5288" href="Agda.Builtin.Bool.html#192" class="InductiveConstructor">false</a> <a id="5294" href="Agda.Builtin.Bool.html#198" class="InductiveConstructor">true</a>  <a id="5300" class="Symbol">=</a> <a id="5302" href="Data.Sum.Base.html#675" class="InductiveConstructor">inj₁</a> <a id="5307" class="Symbol">λ</a> <a id="5309" class="Symbol">()</a>
<a id="5312" href="Chapter.Logic.Negation.html#5132" class="Function">Bool-eq-decidable</a> <a id="5330" href="Agda.Builtin.Bool.html#192" class="InductiveConstructor">false</a> <a id="5336" href="Agda.Builtin.Bool.html#192" class="InductiveConstructor">false</a> <a id="5342" class="Symbol">=</a> <a id="5344" href="Data.Sum.Base.html#700" class="InductiveConstructor">inj₂</a> <a id="5349" href="Agda.Builtin.Equality.html#207" class="InductiveConstructor">refl</a>
</pre>
Note that we use the constructor `inj₂` for representing a positive
answer to the question "is `x` equal to `y`?" and `inj₁` for
representing a negative answer. For readability purposes, it may be
appropriate to give these constructors more evocative names, such as
`yes` and `no`. We can do so (without defining an *ad hoc*
`Decidable` data type) by means of **pattern synonyms**.

<pre class="Agda"><a id="5746" class="Keyword">pattern</a> <a id="yes"></a><a id="5754" href="Chapter.Logic.Negation.html#5754" class="InductiveConstructor">yes</a> <a id="5758" href="Chapter.Logic.Negation.html#5767" class="Bound">x</a> <a id="5760" class="Symbol">=</a> <a id="5762" href="Data.Sum.Base.html#700" class="InductiveConstructor">inj₂</a> <a id="5767" href="Chapter.Logic.Negation.html#5767" class="Bound">x</a>
<a id="5769" class="Keyword">pattern</a> <a id="no"></a><a id="5777" href="Chapter.Logic.Negation.html#5777" class="InductiveConstructor">no</a>  <a id="5781" href="Chapter.Logic.Negation.html#5790" class="Bound">x</a> <a id="5783" class="Symbol">=</a> <a id="5785" href="Data.Sum.Base.html#675" class="InductiveConstructor">inj₁</a> <a id="5790" href="Chapter.Logic.Negation.html#5790" class="Bound">x</a>

Another example of decidabile property is the equality for natural
numbers. In this case, when we compare two numbers of the form `suc
x` and `suc y`, we first decide whether `x` and `y` are equal. If
they are not, then we conclude that `suc x` and `suc y` must be
different (recall that constructors such as `suc` are injective). If
`x` and `y` are equal, then we can prove `suc x ≡ suc y` by
congruence.

original lists are equal.
