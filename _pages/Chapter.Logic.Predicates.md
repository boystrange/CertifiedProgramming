## The half of a natural number

Consider the following function which computes the (truncated) half
of a natural number:

<a id="670" href="Chapter.Logic.Predicates.html#656" class="Function">half</a> <a id="675" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a>            <a id="691" class="Symbol">=</a> <a id="693" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a>
<a id="698" href="Chapter.Logic.Predicates.html#656" class="Function">half</a> <a id="703" class="Symbol">(</a><a id="704" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="708" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a><a id="712" class="Symbol">)</a>     <a id="718" class="Symbol">=</a> <a id="720" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a>
<a id="725" href="Chapter.Logic.Predicates.html#656" class="Function">half</a> <a id="730" class="Symbol">(</a><a id="731" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="735" class="Symbol">(</a><a id="736" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="740" href="Chapter.Logic.Predicates.html#740" class="Bound">x</a><a id="741" class="Symbol">))</a> <a id="744" class="Symbol">=</a> <a id="746" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="750" class="Symbol">(</a><a id="751" href="Chapter.Logic.Predicates.html#656" class="Function">half</a> <a id="756" href="Chapter.Logic.Predicates.html#740" class="Bound">x</a><a id="757" class="Symbol">)</a>
</pre>
We would like to prove a theorem asserting that, by doubling the
half of an even natural number `x`, we obtain the original number
`x`. In order to do so, we have to teach Agda what it means for a
natural number to be even. As we will see, there are several ways of
doing this. For each way, we will show how to prove the desired
result discussing pros and cons of the approach.

## Being even, programmatically

The first, and possibly most obvious, way of defining "evenness" is
by means of a recursive function that returns either `true` or
`false`. We call this the "programmer's" way of defining evenness,
since it is the approach that any (functional) programmer would
choose in the first place.

<pre class="Agda"><a id="Even-p"></a><a id="1471" href="Chapter.Logic.Predicates.html#1471" class="Function">Even-p</a> <a id="1478" class="Symbol">:</a> <a id="1480" href="Agda.Builtin.Nat.html#203" class="Datatype">ℕ</a> <a id="1482" class="Symbol">-&gt;</a> <a id="1485" href="Agda.Builtin.Bool.html#173" class="Datatype">Bool</a>
<a id="1490" href="Chapter.Logic.Predicates.html#1471" class="Function">Even-p</a> <a id="1497" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a>            <a id="1513" class="Symbol">=</a> <a id="1515" href="Agda.Builtin.Bool.html#198" class="InductiveConstructor">true</a>
<a id="1520" href="Chapter.Logic.Predicates.html#1471" class="Function">Even-p</a> <a id="1527" class="Symbol">(</a><a id="1528" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="1532" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a><a id="1536" class="Symbol">)</a>     <a id="1542" class="Symbol">=</a> <a id="1544" href="Agda.Builtin.Bool.html#192" class="InductiveConstructor">false</a>
<a id="1550" href="Chapter.Logic.Predicates.html#1471" class="Function">Even-p</a> <a id="1557" class="Symbol">(</a><a id="1558" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="1562" class="Symbol">(</a><a id="1563" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="1567" href="Chapter.Logic.Predicates.html#1567" class="Bound">x</a><a id="1568" class="Symbol">))</a> <a id="1571" class="Symbol">=</a> <a id="1573" href="Chapter.Logic.Predicates.html#1471" class="Function">Even-p</a> <a id="1580" href="Chapter.Logic.Predicates.html#1567" class="Bound">x</a>
</pre>
Notice that `Even-p x` is a *term* of type `Bool`. So, in order to
use `Even-p` as a predicate, we have to compare the result of
`Even-p x` to `true` using equality. We can thus formulate our
theorem as follows.

<pre class="Agda"><a id="theorem-p"></a><a id="1804" href="Chapter.Logic.Predicates.html#1804" class="Function">theorem-p</a> <a id="1814" class="Symbol">:</a> <a id="1816" class="Symbol">∀{</a><a id="1818" href="Chapter.Logic.Predicates.html#1818" class="Bound">x</a> <a id="1820" class="Symbol">:</a> <a id="1822" href="Agda.Builtin.Nat.html#203" class="Datatype">ℕ</a><a id="1823" class="Symbol">}</a> <a id="1825" class="Symbol">(</a><a id="1826" href="Chapter.Logic.Predicates.html#1826" class="Bound">ev</a> <a id="1829" class="Symbol">:</a> <a id="1831" href="Chapter.Logic.Predicates.html#1471" class="Function">Even-p</a> <a id="1838" href="Chapter.Logic.Predicates.html#1818" class="Bound">x</a> <a id="1840" href="Agda.Builtin.Equality.html#150" class="Datatype Operator">≡</a> <a id="1842" href="Agda.Builtin.Bool.html#198" class="InductiveConstructor">true</a><a id="1846" class="Symbol">)</a> <a id="1848" class="Symbol">-&gt;</a> <a id="1851" href="Chapter.Logic.Predicates.html#1818" class="Bound">x</a> <a id="1853" href="Agda.Builtin.Equality.html#150" class="Datatype Operator">≡</a> <a id="1855" href="Chapter.Logic.Predicates.html#656" class="Function">half</a> <a id="1860" href="Chapter.Logic.Predicates.html#1818" class="Bound">x</a> <a id="1862" href="Agda.Builtin.Nat.html#539" class="Primitive Operator">*</a> <a id="1864" class="Number">2</a>
<a id="1866" href="Chapter.Logic.Predicates.html#1804" class="Function">theorem-p</a> <a id="1876" class="Symbol">{</a><a id="1877" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a><a id="1881" class="Symbol">}</a>          <a id="1892" href="Agda.Builtin.Equality.html#207" class="InductiveConstructor">refl</a> <a id="1897" class="Symbol">=</a> <a id="1899" href="Agda.Builtin.Equality.html#207" class="InductiveConstructor">refl</a>
<a id="1904" href="Chapter.Logic.Predicates.html#1804" class="Function">theorem-p</a> <a id="1914" class="Symbol">{</a><a id="1915" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="1919" class="Symbol">(</a><a id="1920" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="1924" href="Chapter.Logic.Predicates.html#1924" class="Bound">x</a><a id="1925" class="Symbol">)}</a> <a id="1928" href="Chapter.Logic.Predicates.html#1928" class="Bound">ev</a>   <a id="1933" class="Symbol">=</a> <a id="1935" href="Relation.Binary.PropositionalEquality.Core.html#1481" class="Function">cong</a> <a id="1940" class="Symbol">(</a><a id="1941" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="1945" href="Function.Base.html#1134" class="Function Operator">∘</a> <a id="1947" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a><a id="1950" class="Symbol">)</a> <a id="1952" class="Symbol">(</a><a id="1953" href="Chapter.Logic.Predicates.html#1804" class="Function">theorem-p</a> <a id="1963" href="Chapter.Logic.Predicates.html#1928" class="Bound">ev</a><a id="1965" class="Symbol">)</a>
</pre>
We are forced to perform case analysis on the (implicit) argument
`x`, since the proof `ev` that `x` is an even number (which must
have the form `refl` for this is the only normal proof of an
equality proof) bears no structure that helps us proving the
theorem. Interestingly, Agda does not propose an equation for the
case `suc zero`. This happens because `Even-p 1` yields `false`,
which is certainly different from `true`, so Agda realizes that this
case is impossible.

## Being even, mathematically

A mathematician asked to define evenness could possibly say that a
natural number `x` is even if it is the double of another natural
number `y`. Being the double of some natural number `y` is a
property that we can specify in Agda using an existential type. So,
we can define this notion of evenness as follows.

<pre class="Agda"><a id="Even-m"></a><a id="2794" href="Chapter.Logic.Predicates.html#2794" class="Function">Even-m</a> <a id="2801" class="Symbol">:</a> <a id="2803" href="Agda.Builtin.Nat.html#203" class="Datatype">ℕ</a> <a id="2805" class="Symbol">-&gt;</a> <a id="2808" href="Agda.Primitive.html#388" class="Primitive">Set</a>
<a id="2812" href="Chapter.Logic.Predicates.html#2794" class="Function">Even-m</a> <a id="2819" href="Chapter.Logic.Predicates.html#2819" class="Bound">x</a> <a id="2821" class="Symbol">=</a> <a id="2823" href="Data.Product.Base.html#1371" class="Function">∃[</a> <a id="2826" href="Chapter.Logic.Predicates.html#2826" class="Bound">y</a> <a id="2828" href="Data.Product.Base.html#1371" class="Function">]</a> <a id="2830" href="Chapter.Logic.Predicates.html#2819" class="Bound">x</a> <a id="2832" href="Agda.Builtin.Equality.html#150" class="Datatype Operator">≡</a> <a id="2834" href="Chapter.Logic.Predicates.html#2826" class="Bound">y</a> <a id="2836" href="Agda.Builtin.Nat.html#539" class="Primitive Operator">*</a> <a id="2838" class="Number">2</a>
</pre>
Unlike `Even-p`, which returns a *term* (of type `Bool`), `Even-m`
returns a *type* (of type `Set`). This type is inhabited by pairs of
the form `(y , p)` where `y` is some natural number and `p` is a
proof that `x` is the double of `y`. For example, we can prove that
`4` is even as follows.

<pre class="Agda"><a id="3143" href="Chapter.Logic.Predicates.html#3143" class="Function">_</a> <a id="3145" class="Symbol">:</a> <a id="3147" href="Chapter.Logic.Predicates.html#2794" class="Function">Even-m</a> <a id="3154" class="Number">4</a>
<a id="3156" class="Symbol">_</a> <a id="3158" class="Symbol">=</a> <a id="3160" class="Number">2</a> <a id="3162" href="Agda.Builtin.Sigma.html#235" class="InductiveConstructor Operator">,</a> <a id="3164" href="Agda.Builtin.Equality.html#207" class="InductiveConstructor">refl</a>
</pre>
Analogously we can show that `1` is *not* even by proving a
contradiction if we assume that it is even.

<pre class="Agda"><a id="3283" href="Chapter.Logic.Predicates.html#3283" class="Function">_</a> <a id="3285" class="Symbol">:</a> <a id="3287" href="Relation.Nullary.Negation.Core.html#677" class="Function Operator">¬</a> <a id="3289" href="Chapter.Logic.Predicates.html#2794" class="Function">Even-m</a> <a id="3296" class="Number">1</a>
<a id="3298" class="Symbol">_</a> <a id="3300" class="Symbol">=</a> <a id="3302" class="Symbol">λ</a> <a id="3304" class="Symbol">{</a> <a id="3306" class="Symbol">(</a><a id="3307" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a> <a id="3312" href="Agda.Builtin.Sigma.html#235" class="InductiveConstructor Operator">,</a> <a id="3314" class="Symbol">())</a> <a id="3318" class="Symbol">;</a> <a id="3320" class="Symbol">(</a><a id="3321" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="3325" class="Symbol">_</a> <a id="3327" href="Agda.Builtin.Sigma.html#235" class="InductiveConstructor Operator">,</a> <a id="3329" class="Symbol">())</a> <a id="3333" class="Symbol">}</a>
</pre>
When proving that doubling the half of an even number `x` yields
`x`, we can perform case analysis on the proof that `x` is even
obtaining some witness `y` whose double is known to be `x`. We can
conclude if we are able to show that halving a doubled number `y`
yields `y`. For this, we need to prove an auxiliary lemma, which we
locally define within `theorem-m` after the keyword `where`.

## Type-level computations

In the previous approach we have defined a function `Even-m` that,
applied to a natural number `x`, yields the type of proofs that `x`
is even. We can apply the same principle to write an alternative
version of `Even-p` in which the result is a type instead of a
boolean value. In this case, the returned type is `⊤` when we
realize that `x` is even and `⊥` otherwise. This approach is
sometimes referred to as making use of *type-level computations*
because it computes a type (`Even-r x`) from a term `x` (technically
speaking, also `Even-m` is defined in this way).

<pre class="Agda"><a id="Even-r"></a><a id="4546" href="Chapter.Logic.Predicates.html#4546" class="Function">Even-r</a> <a id="4553" class="Symbol">:</a> <a id="4555" href="Agda.Builtin.Nat.html#203" class="Datatype">ℕ</a> <a id="4557" class="Symbol">-&gt;</a> <a id="4560" href="Agda.Primitive.html#388" class="Primitive">Set</a>
<a id="4564" href="Chapter.Logic.Predicates.html#4546" class="Function">Even-r</a> <a id="4571" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a>            <a id="4587" class="Symbol">=</a> <a id="4589" href="Agda.Builtin.Unit.html#175" class="Record">⊤</a>
<a id="4591" href="Chapter.Logic.Predicates.html#4546" class="Function">Even-r</a> <a id="4598" class="Symbol">(</a><a id="4599" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="4603" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a><a id="4607" class="Symbol">)</a>     <a id="4613" class="Symbol">=</a> <a id="4615" href="Data.Empty.html#914" class="Function">⊥</a>
<a id="4617" href="Chapter.Logic.Predicates.html#4546" class="Function">Even-r</a> <a id="4624" class="Symbol">(</a><a id="4625" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="4629" class="Symbol">(</a><a id="4630" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="4634" href="Chapter.Logic.Predicates.html#4634" class="Bound">x</a><a id="4635" class="Symbol">))</a> <a id="4638" class="Symbol">=</a> <a id="4640" href="Chapter.Logic.Predicates.html#4546" class="Function">Even-r</a> <a id="4647" href="Chapter.Logic.Predicates.html#4634" class="Bound">x</a>
</pre>
Compared to `Even-p`, the advantage of `Even-r` is that it yields a
type, so it is not necessary to use equality to turn `Even-r x` into
a type. However, just like when using `Even-p`, the proof of `Even-r
x` is simply `<>`, so inspecting the proof of `Even-r x` does not
reveal anything useful about `x` and we are forced to perform case

## Being even as an inference system

The final point of view we consider is motivated by the observation
that the set of *even natural numbers* is the smallest set $X$ such
that:

* $0 \in X$, and
* if $x \in X$, then $2 + x \in X$ as well.

In other words, we can characterize the whole set of even numbers as
those satisfying the predicate `Even x`, where `Even` is inductively
defined by the following inference rules.

                                         Even x
    [even-zero] ------    [even-succ] ------------
                Even 0                Even (2 + x)

The axiom `[even-zero]` asserts that `0` is an even number. The rule
`[even-succ]` asserts that `2 + x` is even whenever `x` is. We can
define this inference system as an inductive data type such that

* the name of the data type (`Even-i`) corresponds to the name of
  the predicate (`Even`) we are defining;
* the constructors of the data type correspond to axioms/rules in
  the inference system defining the predicate;
* terms of the data type correspond to derivations in this inference system.

Note that the evenness predicate `Even-i x` we are defining in this
way depends on the natural number `x` that is claimed to be even. We
cannot express this dependency merely using a parameter of the data
type, since parameters are supposed to be the same across the whole
data type definition whereas the value of `x` varies (e.g., it is
`0` in `[even-zero]` and it is `x` and `2 + x` in
`[even-succ]`). For this reason, we define an **indexed data type**
(also known as **inductive family**), which differs from a plain (or
parametric) data type as it contains one or more **indexes**. In our
case, a single index of type `ℕ` suffices.

## Exercises

1. Prove that all the provided definitions of evenness are
   equivalent, for instance that `Even-p x ≡ true` implies `Even-r
   x`, that `Even-r x` implies `Even-i x`, that `Even-i x` implies
   `Even-m x` and that `Even-m x` implies `Even-p x ≡ true`.
2. Prove that `x ≡ 1 + x /2 * 2` when `¬ Even-i x` holds.
3. Prove that `Even-i` is decidable, namely the theorem `∀(x : ℕ) ->
   Decidable (Even-i x)`.
4. Define an indexed data type `Odd-i` analogous to `Even-i` but
   such that `Odd-i x` holds if and only if `x` is odd. Prove that
   `5` is odd and `2` is not.
5. Prove that `Even-i x ∨ Odd-i x` holds and that `Even-i x ∧ Odd-i
   x` does not for every `x`.
6. Prove that `Odd-i x` implies `x ≡ 1 + x/2 * 2` without using
   recursion, but reusing the results of exercises 2 and 4.
