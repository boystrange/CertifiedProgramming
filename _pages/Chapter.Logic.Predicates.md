## The half of a natural number

Consider the following function which computes the (truncated) half
of a natural number:

<pre class="Agda"><a id="half"></a><a id="656" href="Chapter.Logic.Predicates.html#656" class="Function">half</a> <a id="661" class="Symbol">:</a> <a id="663" href="Agda.Builtin.Nat.html#203" class="Datatype">ℕ</a> <a id="665" class="Symbol">→</a> <a id="667" href="Agda.Builtin.Nat.html#203" class="Datatype">ℕ</a>
<a id="669" href="Chapter.Logic.Predicates.html#656" class="Function">half</a> <a id="674" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a>          <a id="688" class="Symbol">=</a> <a id="690" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a>
<a id="695" href="Chapter.Logic.Predicates.html#656" class="Function">half</a> <a id="700" class="Symbol">(</a><a id="701" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="705" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a><a id="709" class="Symbol">)</a>    <a id="714" class="Symbol">=</a> <a id="716" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a>
<a id="721" href="Chapter.Logic.Predicates.html#656" class="Function">half</a> <a id="726" class="Symbol">(</a><a id="727" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="731" class="Symbol">(</a><a id="732" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="736" href="Chapter.Logic.Predicates.html#736" class="Bound">x</a><a id="737" class="Symbol">))</a> <a id="740" class="Symbol">=</a> <a id="742" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="746" class="Symbol">(</a><a id="747" href="Chapter.Logic.Predicates.html#656" class="Function">half</a> <a id="752" href="Chapter.Logic.Predicates.html#736" class="Bound">x</a><a id="753" class="Symbol">)</a>
</pre>
We would like to prove a theorem asserting that, by doubling the
half of an even natural number `x`, we obtain the original number
`x`. In order to do so, we have to teach Agda what it means for a
natural number to be even. As we will see, there are several ways of
doing this. For each way, we will show how to prove the desired
result discussing pros and cons of the approach.

## Being even, programmatically

The first, and possibly most obvious, way of defining "evenness" is
by means of a recursive function that returns either `true` or
`false`. We call this the "programmer's" way of defining evenness,
since it is the approach that any (functional) programmer would
choose in the first place.

<pre class="Agda"><a id="Even-p"></a><a id="1467" href="Chapter.Logic.Predicates.html#1467" class="Function">Even-p</a> <a id="1474" class="Symbol">:</a> <a id="1476" href="Agda.Builtin.Nat.html#203" class="Datatype">ℕ</a> <a id="1478" class="Symbol">→</a> <a id="1480" href="Agda.Builtin.Bool.html#173" class="Datatype">Bool</a>
<a id="1485" href="Chapter.Logic.Predicates.html#1467" class="Function">Even-p</a> <a id="1492" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a>          <a id="1506" class="Symbol">=</a> <a id="1508" href="Agda.Builtin.Bool.html#198" class="InductiveConstructor">true</a>
<a id="1513" href="Chapter.Logic.Predicates.html#1467" class="Function">Even-p</a> <a id="1520" class="Symbol">(</a><a id="1521" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="1525" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a><a id="1529" class="Symbol">)</a>    <a id="1534" class="Symbol">=</a> <a id="1536" href="Agda.Builtin.Bool.html#192" class="InductiveConstructor">false</a>
<a id="1542" href="Chapter.Logic.Predicates.html#1467" class="Function">Even-p</a> <a id="1549" class="Symbol">(</a><a id="1550" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="1554" class="Symbol">(</a><a id="1555" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="1559" href="Chapter.Logic.Predicates.html#1559" class="Bound">x</a><a id="1560" class="Symbol">))</a> <a id="1563" class="Symbol">=</a> <a id="1565" href="Chapter.Logic.Predicates.html#1467" class="Function">Even-p</a> <a id="1572" href="Chapter.Logic.Predicates.html#1559" class="Bound">x</a>
</pre>
Notice that `Even-p x` is a *term* of type `Bool`. So, in order to
use `Even-p` as a predicate, we have to compare the result of
`Even-p x` to `true` using equality. We can thus formulate our
theorem as follows.

<pre class="Agda"><a id="theorem-p"></a><a id="1796" href="Chapter.Logic.Predicates.html#1796" class="Function">theorem-p</a> <a id="1806" class="Symbol">:</a> <a id="1808" class="Symbol">∀{</a><a id="1810" href="Chapter.Logic.Predicates.html#1810" class="Bound">x</a> <a id="1812" class="Symbol">:</a> <a id="1814" href="Agda.Builtin.Nat.html#203" class="Datatype">ℕ</a><a id="1815" class="Symbol">}</a> <a id="1817" class="Symbol">→</a> <a id="1819" href="Chapter.Logic.Predicates.html#1467" class="Function">Even-p</a> <a id="1826" href="Chapter.Logic.Predicates.html#1810" class="Bound">x</a> <a id="1828" href="Agda.Builtin.Equality.html#150" class="Datatype Operator">≡</a> <a id="1830" href="Agda.Builtin.Bool.html#198" class="InductiveConstructor">true</a> <a id="1835" class="Symbol">→</a> <a id="1837" href="Chapter.Logic.Predicates.html#1810" class="Bound">x</a> <a id="1839" href="Agda.Builtin.Equality.html#150" class="Datatype Operator">≡</a> <a id="1841" href="Chapter.Logic.Predicates.html#656" class="Function">half</a> <a id="1846" href="Chapter.Logic.Predicates.html#1810" class="Bound">x</a> <a id="1848" href="Agda.Builtin.Nat.html#539" class="Primitive Operator">*</a> <a id="1850" class="Number">2</a>
<a id="1852" href="Chapter.Logic.Predicates.html#1796" class="Function">theorem-p</a> <a id="1862" class="Symbol">{</a><a id="1863" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a><a id="1867" class="Symbol">}</a>        <a id="1876" href="Agda.Builtin.Equality.html#207" class="InductiveConstructor">refl</a> <a id="1881" class="Symbol">=</a> <a id="1883" href="Agda.Builtin.Equality.html#207" class="InductiveConstructor">refl</a>
<a id="1888" href="Chapter.Logic.Predicates.html#1796" class="Function">theorem-p</a> <a id="1898" class="Symbol">{</a><a id="1899" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="1903" class="Symbol">(</a><a id="1904" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="1908" href="Chapter.Logic.Predicates.html#1908" class="Bound">x</a><a id="1909" class="Symbol">)}</a> <a id="1912" href="Chapter.Logic.Predicates.html#1912" class="Bound">ev</a>   <a id="1917" class="Symbol">=</a> <a id="1919" href="Relation.Binary.PropositionalEquality.Core.html#1481" class="Function">cong</a> <a id="1924" class="Symbol">(</a><a id="1925" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="1929" href="Function.Base.html#1134" class="Function Operator">∘</a> <a id="1931" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a><a id="1934" class="Symbol">)</a> <a id="1936" class="Symbol">(</a><a id="1937" href="Chapter.Logic.Predicates.html#1796" class="Function">theorem-p</a> <a id="1947" href="Chapter.Logic.Predicates.html#1912" class="Bound">ev</a><a id="1949" class="Symbol">)</a>
</pre>
We are forced to perform case analysis on the (implicit) argument
`x`, since the proof `ev` that `x` is an even number (which must
have the form `refl` for this is the only normal proof of an
equality proof) bears no structure that helps us proving the
theorem. Interestingly, Agda does not propose an equation for the
case `suc zero`. This happens because `Even-p 1` yields `false`,
which is certainly different from `true`, so Agda realizes that this
case is impossible.

## Being even, mathematically

A mathematician asked to define evenness could possibly say that a
natural number `x` is even if it is the double of another natural
number `y`. Being the double of some natural number `y` is a
property that we can specify in Agda using an existential type. So,
we can define this notion of evenness as follows.

<pre class="Agda"><a id="Even-m"></a><a id="2778" href="Chapter.Logic.Predicates.html#2778" class="Function">Even-m</a> <a id="2785" class="Symbol">:</a> <a id="2787" href="Agda.Builtin.Nat.html#203" class="Datatype">ℕ</a> <a id="2789" class="Symbol">→</a> <a id="2791" href="Agda.Primitive.html#388" class="Primitive">Set</a>
<a id="2795" href="Chapter.Logic.Predicates.html#2778" class="Function">Even-m</a> <a id="2802" href="Chapter.Logic.Predicates.html#2802" class="Bound">x</a> <a id="2804" class="Symbol">=</a> <a id="2806" href="Data.Product.Base.html#1371" class="Function">∃[</a> <a id="2809" href="Chapter.Logic.Predicates.html#2809" class="Bound">y</a> <a id="2811" href="Data.Product.Base.html#1371" class="Function">]</a> <a id="2813" href="Chapter.Logic.Predicates.html#2802" class="Bound">x</a> <a id="2815" href="Agda.Builtin.Equality.html#150" class="Datatype Operator">≡</a> <a id="2817" href="Chapter.Logic.Predicates.html#2809" class="Bound">y</a> <a id="2819" href="Agda.Builtin.Nat.html#539" class="Primitive Operator">*</a> <a id="2821" class="Number">2</a>
</pre>
Unlike `Even-p`, which returns a *term* (of type `Bool`), `Even-m`
returns a *type* (of type `Set`). This type is inhabited by pairs of
the form `(y , p)` where `y` is some natural number and `p` is a
proof that `x` is the double of `y`. For example, we can prove that
`4` is even as follows.

<pre class="Agda"><a id="3126" href="Chapter.Logic.Predicates.html#3126" class="Function">_</a> <a id="3128" class="Symbol">:</a> <a id="3130" href="Chapter.Logic.Predicates.html#2778" class="Function">Even-m</a> <a id="3137" class="Number">4</a>
<a id="3139" class="Symbol">_</a> <a id="3141" class="Symbol">=</a> <a id="3143" class="Number">2</a> <a id="3145" href="Agda.Builtin.Sigma.html#235" class="InductiveConstructor Operator">,</a> <a id="3147" href="Agda.Builtin.Equality.html#207" class="InductiveConstructor">refl</a>
</pre>
Analogously we can show that `1` is *not* even by proving a
contradiction if we assume that it is even.

<pre class="Agda"><a id="3266" href="Chapter.Logic.Predicates.html#3266" class="Function">_</a> <a id="3268" class="Symbol">:</a> <a id="3270" href="Relation.Nullary.Negation.Core.html#677" class="Function Operator">¬</a> <a id="3272" href="Chapter.Logic.Predicates.html#2778" class="Function">Even-m</a> <a id="3279" class="Number">1</a>
<a id="3281" class="Symbol">_</a> <a id="3283" class="Symbol">=</a> <a id="3285" class="Symbol">λ</a> <a id="3287" class="Symbol">{</a> <a id="3289" class="Symbol">(</a><a id="3290" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a> <a id="3295" href="Agda.Builtin.Sigma.html#235" class="InductiveConstructor Operator">,</a> <a id="3297" class="Symbol">())</a> <a id="3301" class="Symbol">;</a> <a id="3303" class="Symbol">(</a><a id="3304" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="3308" class="Symbol">_</a> <a id="3310" href="Agda.Builtin.Sigma.html#235" class="InductiveConstructor Operator">,</a> <a id="3312" class="Symbol">())</a> <a id="3316" class="Symbol">}</a>
</pre>
When proving that doubling the half of an even number `x` yields
`x`, we can perform case analysis on the proof that `x` is even
obtaining some witness `y` whose double is known to be `x`. We can
conclude if we are able to show that halving a doubled number `y`
yields `y`. For this, we need to prove an auxiliary lemma, which we
locally define within `theorem-m` after the keyword `where`.

## Type-level computations

In the previous approach we have defined a function `Even-m` that,
applied to a natural number `x`, yields the type of proofs that `x`
is even. We can apply the same principle to write an alternative
version of `Even-p` in which the result is a type instead of a
boolean value. In this case, the returned type is `⊤` when we
realize that `x` is even and `⊥` otherwise. This approach is
sometimes referred to as making use of *type-level computations*
because it computes a type (`Even-r x`) from a term `x` (technically
speaking, also `Even-m` is defined in this way).

<pre class="Agda"><a id="Even-r"></a><a id="4522" href="Chapter.Logic.Predicates.html#4522" class="Function">Even-r</a> <a id="4529" class="Symbol">:</a> <a id="4531" href="Agda.Builtin.Nat.html#203" class="Datatype">ℕ</a> <a id="4533" class="Symbol">→</a> <a id="4535" href="Agda.Primitive.html#388" class="Primitive">Set</a>
<a id="4539" href="Chapter.Logic.Predicates.html#4522" class="Function">Even-r</a> <a id="4546" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a>            <a id="4562" class="Symbol">=</a> <a id="4564" href="Agda.Builtin.Unit.html#175" class="Record">⊤</a>
<a id="4566" href="Chapter.Logic.Predicates.html#4522" class="Function">Even-r</a> <a id="4573" class="Symbol">(</a><a id="4574" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="4578" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a><a id="4582" class="Symbol">)</a>     <a id="4588" class="Symbol">=</a> <a id="4590" href="Data.Empty.html#914" class="Function">⊥</a>
<a id="4592" href="Chapter.Logic.Predicates.html#4522" class="Function">Even-r</a> <a id="4599" class="Symbol">(</a><a id="4600" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="4604" class="Symbol">(</a><a id="4605" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="4609" href="Chapter.Logic.Predicates.html#4609" class="Bound">x</a><a id="4610" class="Symbol">))</a> <a id="4613" class="Symbol">=</a> <a id="4615" href="Chapter.Logic.Predicates.html#4522" class="Function">Even-r</a> <a id="4622" href="Chapter.Logic.Predicates.html#4609" class="Bound">x</a>
</pre>
Compared to `Even-p`, the advantage of `Even-r` is that it yields a
type, so it is not necessary to use equality to turn `Even-r x` into
a type. However, just like when using `Even-p`, the proof of `Even-r
x` is simply `tt`, so inspecting the proof of `Even-r x` does not
reveal anything useful about `x` and we are forced to perform case
analysis on `x` to complete our theorem.

## Being even as an inference system

The final point of view we consider is motivated by the observation
that the set of *even natural numbers* is the smallest set $X$ such
that:

* $0 \in X$, and
* if $x \in X$, then $2 + x \in X$ as well.

In other words, we can characterize the whole set of even numbers as
those satisfying the predicate `Even x`, where `Even` is inductively
defined by the following inference rules.

                                        Even x
    [even-zero] ------    [even-suc] ------------
                Even 0               Even (2 + x)

The axiom `[even-zero]` asserts that `0` is an even number. The rule
`[even-suc]` asserts that `2 + x` is even whenever `x` is. We can
define this inference system as an inductive data type such that

* the name of the data type (`Even-i`) corresponds to the name of
  the predicate (`Even`) we are defining;
* the constructors of the data type correspond to axioms/rules in
  the inference system defining the predicate;
* terms of the data type correspond to derivations in this inference system.

Note that the evenness predicate `Even-i x` we are defining in this
way depends on the natural number `x` that is claimed to be even. We
cannot express this dependency merely using a parameter of the data
type, since parameters are supposed to be the same across the whole
data type definition whereas the value of `x` varies (e.g., it is
`0` in `[even-zero]` and it is `x` and `2 + x` in
`[even-suc]`). For this reason, we define an **indexed data type**
(also known as **inductive family**), which differs from a plain (or
parametric) data type as it contains one or more **indexes**. In our
case, a single index of type `ℕ` suffices.

of building terms of type `Even-i x`. One is through the constructor
`even-zero`. In this case `x` must be `0`. The other is through the
constructor `even-suc` applied to a term of type `Even-i x`, which
yields a term of type `Even-i (2 + x)`.

## Exercises

1. Prove that all the provided definitions of evenness are
   equivalent, for instance that `Even-p x ≡ true` implies `Even-r
   x`, that `Even-r x` implies `Even-i x`, that `Even-i x` implies
   `Even-m x` and that `Even-m x` implies `Even-p x ≡ true`.
2. Prove that `x ≡ 1 + x /2 * 2` when `¬ Even-i x` holds.
3. Prove that `Even-i` is decidable, namely the theorem `∀(x : ℕ) →
   ¬ Even-i x ⊎ Even-i x`.
4. Define an indexed data type `Odd-i` analogous to `Even-i` but
   such that `Odd-i x` holds if and only if `x` is odd. Prove that
   `5` is odd and `2` is not.
5. Prove that `Even-i x ⊎ Odd-i x` holds and that `Even-i x × Odd-i
   x` does not for every `x`.
6. Prove that `Odd-i x` implies `x ≡ 1 + x/2 * 2` without using
   recursion, but reusing the results of exercises 2 and 4.
