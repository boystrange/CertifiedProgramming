## The half of a natural number

Consider the following function which computes the (truncated) half
of a natural number:

<a id="670" href="Chapter.Logic.Predicates.html#656" class="Function">half</a> <a id="675" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a>            <a id="691" class="Symbol">=</a> <a id="693" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a>
<a id="698" href="Chapter.Logic.Predicates.html#656" class="Function">half</a> <a id="703" class="Symbol">(</a><a id="704" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="708" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a><a id="712" class="Symbol">)</a>     <a id="718" class="Symbol">=</a> <a id="720" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a>
<a id="725" href="Chapter.Logic.Predicates.html#656" class="Function">half</a> <a id="730" class="Symbol">(</a><a id="731" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="735" class="Symbol">(</a><a id="736" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="740" href="Chapter.Logic.Predicates.html#740" class="Bound">x</a><a id="741" class="Symbol">))</a> <a id="744" class="Symbol">=</a> <a id="746" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="750" class="Symbol">(</a><a id="751" href="Chapter.Logic.Predicates.html#656" class="Function">half</a> <a id="756" href="Chapter.Logic.Predicates.html#740" class="Bound">x</a><a id="757" class="Symbol">)</a>
</pre>
We would like to prove a theorem asserting that, by doubling the
half of an even natural number `x`, we obtain the original number
`x`. In order to do so, we have to teach Agda what it means for a
natural number to be even. As we will see, there are several ways of
doing this. For each way, we will show how to prove the desired
result discussing pros and cons of the approach.

## Being even, programmatically

The first, and possibly most obvious, way of defining "evenness" is
by means of a recursive function that returns either `true` or
`false`. We call this the "programmer's" way of defining evenness,
since it is the approach that any (functional) programmer would
choose in the first place.

<pre class="Agda"><a id="Even-p"></a><a id="1471" href="Chapter.Logic.Predicates.html#1471" class="Function">Even-p</a> <a id="1478" class="Symbol">:</a> <a id="1480" href="Agda.Builtin.Nat.html#203" class="Datatype">ℕ</a> <a id="1482" class="Symbol">-&gt;</a> <a id="1485" href="Agda.Builtin.Bool.html#173" class="Datatype">Bool</a>
<a id="1490" href="Chapter.Logic.Predicates.html#1471" class="Function">Even-p</a> <a id="1497" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a>          <a id="1511" class="Symbol">=</a> <a id="1513" href="Agda.Builtin.Bool.html#198" class="InductiveConstructor">true</a>
<a id="1518" href="Chapter.Logic.Predicates.html#1471" class="Function">Even-p</a> <a id="1525" class="Symbol">(</a><a id="1526" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="1530" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a><a id="1534" class="Symbol">)</a>    <a id="1539" class="Symbol">=</a> <a id="1541" href="Agda.Builtin.Bool.html#192" class="InductiveConstructor">false</a>
<a id="1547" href="Chapter.Logic.Predicates.html#1471" class="Function">Even-p</a> <a id="1554" class="Symbol">(</a><a id="1555" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="1559" class="Symbol">(</a><a id="1560" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="1564" href="Chapter.Logic.Predicates.html#1564" class="Bound">x</a><a id="1565" class="Symbol">))</a> <a id="1568" class="Symbol">=</a> <a id="1570" href="Chapter.Logic.Predicates.html#1471" class="Function">Even-p</a> <a id="1577" href="Chapter.Logic.Predicates.html#1564" class="Bound">x</a>
</pre>
Notice that `Even-p x` is a *term* of type `Bool`. So, in order to
use `Even-p` as a predicate, we have to compare the result of
`Even-p x` to `true` using equality. We can thus formulate our
theorem as follows.

<pre class="Agda"><a id="theorem-p"></a><a id="1801" href="Chapter.Logic.Predicates.html#1801" class="Function">theorem-p</a> <a id="1811" class="Symbol">:</a> <a id="1813" class="Symbol">∀{</a><a id="1815" href="Chapter.Logic.Predicates.html#1815" class="Bound">x</a> <a id="1817" class="Symbol">:</a> <a id="1819" href="Agda.Builtin.Nat.html#203" class="Datatype">ℕ</a><a id="1820" class="Symbol">}</a> <a id="1822" class="Symbol">(</a><a id="1823" href="Chapter.Logic.Predicates.html#1823" class="Bound">ev</a> <a id="1826" class="Symbol">:</a> <a id="1828" href="Chapter.Logic.Predicates.html#1471" class="Function">Even-p</a> <a id="1835" href="Chapter.Logic.Predicates.html#1815" class="Bound">x</a> <a id="1837" href="Agda.Builtin.Equality.html#150" class="Datatype Operator">≡</a> <a id="1839" href="Agda.Builtin.Bool.html#198" class="InductiveConstructor">true</a><a id="1843" class="Symbol">)</a> <a id="1845" class="Symbol">-&gt;</a> <a id="1848" href="Chapter.Logic.Predicates.html#1815" class="Bound">x</a> <a id="1850" href="Agda.Builtin.Equality.html#150" class="Datatype Operator">≡</a> <a id="1852" href="Chapter.Logic.Predicates.html#656" class="Function">half</a> <a id="1857" href="Chapter.Logic.Predicates.html#1815" class="Bound">x</a> <a id="1859" href="Agda.Builtin.Nat.html#539" class="Primitive Operator">*</a> <a id="1861" class="Number">2</a>
<a id="1863" href="Chapter.Logic.Predicates.html#1801" class="Function">theorem-p</a> <a id="1873" class="Symbol">{</a><a id="1874" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a><a id="1878" class="Symbol">}</a>        <a id="1887" href="Agda.Builtin.Equality.html#207" class="InductiveConstructor">refl</a> <a id="1892" class="Symbol">=</a> <a id="1894" href="Agda.Builtin.Equality.html#207" class="InductiveConstructor">refl</a>
<a id="1899" href="Chapter.Logic.Predicates.html#1801" class="Function">theorem-p</a> <a id="1909" class="Symbol">{</a><a id="1910" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="1914" class="Symbol">(</a><a id="1915" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="1919" href="Chapter.Logic.Predicates.html#1919" class="Bound">x</a><a id="1920" class="Symbol">)}</a> <a id="1923" href="Chapter.Logic.Predicates.html#1923" class="Bound">ev</a>   <a id="1928" class="Symbol">=</a> <a id="1930" href="Relation.Binary.PropositionalEquality.Core.html#1481" class="Function">cong</a> <a id="1935" class="Symbol">(</a><a id="1936" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="1940" href="Function.Base.html#1134" class="Function Operator">∘</a> <a id="1942" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a><a id="1945" class="Symbol">)</a> <a id="1947" class="Symbol">(</a><a id="1948" href="Chapter.Logic.Predicates.html#1801" class="Function">theorem-p</a> <a id="1958" href="Chapter.Logic.Predicates.html#1923" class="Bound">ev</a><a id="1960" class="Symbol">)</a>
</pre>
We are forced to perform case analysis on the (implicit) argument
`x`, since the proof `ev` that `x` is an even number (which must
have the form `refl` for this is the only normal proof of an
equality proof) bears no structure that helps us proving the
theorem. Interestingly, Agda does not propose an equation for the
case `suc zero`. This happens because `Even-p 1` yields `false`,
which is certainly different from `true`, so Agda realizes that this
case is impossible.

## Being even, mathematically

A mathematician asked to define evenness could possibly say that a
natural number `x` is even if it is the double of another natural
number `y`. Being the double of some natural number `y` is a
property that we can specify in Agda using an existential type. So,
we can define this notion of evenness as follows.

<pre class="Agda"><a id="Even-m"></a><a id="2789" href="Chapter.Logic.Predicates.html#2789" class="Function">Even-m</a> <a id="2796" class="Symbol">:</a> <a id="2798" href="Agda.Builtin.Nat.html#203" class="Datatype">ℕ</a> <a id="2800" class="Symbol">-&gt;</a> <a id="2803" href="Agda.Primitive.html#388" class="Primitive">Set</a>
<a id="2807" href="Chapter.Logic.Predicates.html#2789" class="Function">Even-m</a> <a id="2814" href="Chapter.Logic.Predicates.html#2814" class="Bound">x</a> <a id="2816" class="Symbol">=</a> <a id="2818" href="Data.Product.Base.html#1371" class="Function">∃[</a> <a id="2821" href="Chapter.Logic.Predicates.html#2821" class="Bound">y</a> <a id="2823" href="Data.Product.Base.html#1371" class="Function">]</a> <a id="2825" href="Chapter.Logic.Predicates.html#2814" class="Bound">x</a> <a id="2827" href="Agda.Builtin.Equality.html#150" class="Datatype Operator">≡</a> <a id="2829" href="Chapter.Logic.Predicates.html#2821" class="Bound">y</a> <a id="2831" href="Agda.Builtin.Nat.html#539" class="Primitive Operator">*</a> <a id="2833" class="Number">2</a>
</pre>
Unlike `Even-p`, which returns a *term* (of type `Bool`), `Even-m`
returns a *type* (of type `Set`). This type is inhabited by pairs of
the form `(y , p)` where `y` is some natural number and `p` is a
proof that `x` is the double of `y`. For example, we can prove that
`4` is even as follows.

<pre class="Agda"><a id="3138" href="Chapter.Logic.Predicates.html#3138" class="Function">_</a> <a id="3140" class="Symbol">:</a> <a id="3142" href="Chapter.Logic.Predicates.html#2789" class="Function">Even-m</a> <a id="3149" class="Number">4</a>
<a id="3151" class="Symbol">_</a> <a id="3153" class="Symbol">=</a> <a id="3155" class="Number">2</a> <a id="3157" href="Agda.Builtin.Sigma.html#235" class="InductiveConstructor Operator">,</a> <a id="3159" href="Agda.Builtin.Equality.html#207" class="InductiveConstructor">refl</a>
</pre>
Analogously we can show that `1` is *not* even by proving a
contradiction if we assume that it is even.

<pre class="Agda"><a id="3278" href="Chapter.Logic.Predicates.html#3278" class="Function">_</a> <a id="3280" class="Symbol">:</a> <a id="3282" href="Relation.Nullary.Negation.Core.html#677" class="Function Operator">¬</a> <a id="3284" href="Chapter.Logic.Predicates.html#2789" class="Function">Even-m</a> <a id="3291" class="Number">1</a>
<a id="3293" class="Symbol">_</a> <a id="3295" class="Symbol">=</a> <a id="3297" class="Symbol">λ</a> <a id="3299" class="Symbol">{</a> <a id="3301" class="Symbol">(</a><a id="3302" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a> <a id="3307" href="Agda.Builtin.Sigma.html#235" class="InductiveConstructor Operator">,</a> <a id="3309" class="Symbol">())</a> <a id="3313" class="Symbol">;</a> <a id="3315" class="Symbol">(</a><a id="3316" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="3320" class="Symbol">_</a> <a id="3322" href="Agda.Builtin.Sigma.html#235" class="InductiveConstructor Operator">,</a> <a id="3324" class="Symbol">())</a> <a id="3328" class="Symbol">}</a>
</pre>
When proving that doubling the half of an even number `x` yields
`x`, we can perform case analysis on the proof that `x` is even
obtaining some witness `y` whose double is known to be `x`. We can
conclude if we are able to show that halving a doubled number `y`
yields `y`. For this, we need to prove an auxiliary lemma, which we
locally define within `theorem-m` after the keyword `where`.

## Type-level computations

In the previous approach we have defined a function `Even-m` that,
applied to a natural number `x`, yields the type of proofs that `x`
is even. We can apply the same principle to write an alternative
version of `Even-p` in which the result is a type instead of a
boolean value. In this case, the returned type is `⊤` when we
realize that `x` is even and `⊥` otherwise. This approach is
sometimes referred to as making use of *type-level computations*
because it computes a type (`Even-r x`) from a term `x` (technically
speaking, also `Even-m` is defined in this way).

<pre class="Agda"><a id="Even-r"></a><a id="4541" href="Chapter.Logic.Predicates.html#4541" class="Function">Even-r</a> <a id="4548" class="Symbol">:</a> <a id="4550" href="Agda.Builtin.Nat.html#203" class="Datatype">ℕ</a> <a id="4552" class="Symbol">-&gt;</a> <a id="4555" href="Agda.Primitive.html#388" class="Primitive">Set</a>
<a id="4559" href="Chapter.Logic.Predicates.html#4541" class="Function">Even-r</a> <a id="4566" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a>            <a id="4582" class="Symbol">=</a> <a id="4584" href="Agda.Builtin.Unit.html#175" class="Record">⊤</a>
<a id="4586" href="Chapter.Logic.Predicates.html#4541" class="Function">Even-r</a> <a id="4593" class="Symbol">(</a><a id="4594" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="4598" href="Agda.Builtin.Nat.html#221" class="InductiveConstructor">zero</a><a id="4602" class="Symbol">)</a>     <a id="4608" class="Symbol">=</a> <a id="4610" href="Data.Empty.html#914" class="Function">⊥</a>
<a id="4612" href="Chapter.Logic.Predicates.html#4541" class="Function">Even-r</a> <a id="4619" class="Symbol">(</a><a id="4620" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="4624" class="Symbol">(</a><a id="4625" href="Agda.Builtin.Nat.html#234" class="InductiveConstructor">suc</a> <a id="4629" href="Chapter.Logic.Predicates.html#4629" class="Bound">x</a><a id="4630" class="Symbol">))</a> <a id="4633" class="Symbol">=</a> <a id="4635" href="Chapter.Logic.Predicates.html#4541" class="Function">Even-r</a> <a id="4642" href="Chapter.Logic.Predicates.html#4629" class="Bound">x</a>
</pre>
Compared to `Even-p`, the advantage of `Even-r` is that it yields a
type, so it is not necessary to use equality to turn `Even-r x` into
a type. However, just like when using `Even-p`, the proof of `Even-r
x` is simply `tt`, so inspecting the proof of `Even-r x` does not
reveal anything useful about `x` and we are forced to perform case

## Being even as an inference system

The final point of view we consider is motivated by the observation
that the set of *even natural numbers* is the smallest set $X$ such
that:

* $0 \in X$, and
* if $x \in X$, then $2 + x \in X$ as well.

In other words, we can characterize the whole set of even numbers as
those satisfying the predicate `Even x`, where `Even` is inductively
defined by the following inference rules.

                                        Even x
    [even-zero] ------    [even-suc] ------------
                Even 0               Even (2 + x)

The axiom `[even-zero]` asserts that `0` is an even number. The rule
`[even-suc]` asserts that `2 + x` is even whenever `x` is. We can
define this inference system as an inductive data type such that

* the name of the data type (`Even-i`) corresponds to the name of
  the predicate (`Even`) we are defining;
* the constructors of the data type correspond to axioms/rules in
  the inference system defining the predicate;
* terms of the data type correspond to derivations in this inference system.

Note that the evenness predicate `Even-i x` we are defining in this
way depends on the natural number `x` that is claimed to be even. We
cannot express this dependency merely using a parameter of the data
type, since parameters are supposed to be the same across the whole
data type definition whereas the value of `x` varies (e.g., it is
`0` in `[even-zero]` and it is `x` and `2 + x` in
`[even-suc]`). For this reason, we define an **indexed data type**
(also known as **inductive family**), which differs from a plain (or
parametric) data type as it contains one or more **indexes**. In our
case, a single index of type `ℕ` suffices.

## Exercises

1. Prove that all the provided definitions of evenness are
   equivalent, for instance that `Even-p x ≡ true` implies `Even-r
   x`, that `Even-r x` implies `Even-i x`, that `Even-i x` implies
   `Even-m x` and that `Even-m x` implies `Even-p x ≡ true`.
2. Prove that `x ≡ 1 + x /2 * 2` when `¬ Even-i x` holds.
3. Prove that `Even-i` is decidable, namely the theorem `∀(x : ℕ) ->
   ¬ Even-i x ⊎ Even-i x`.
4. Define an indexed data type `Odd-i` analogous to `Even-i` but
   such that `Odd-i x` holds if and only if `x` is odd. Prove that
   `5` is odd and `2` is not.
5. Prove that `Even-i x ⊎ Odd-i x` holds and that `Even-i x × Odd-i
   x` does not for every `x`.
6. Prove that `Odd-i x` implies `x ≡ 1 + x/2 * 2` without using
   recursion, but reusing the results of exercises 2 and 4.
